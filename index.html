Fluo incrementally processes large data sets stored in Accumulo

Fluo is an implementation of Percolator for Accumulo that provides the following features:

  • Transactions for concurrent, cross-node updates to data stored in Accumulo
  • Notifications that execute user code when observed data changes
  • A core Fluo API to perform transactional updates using simple get/set methods
  • A higher order Fluo Recipes API that builds on the core Fluo API to support more complex methods for transactional updates

If you are new to Fluo, check out the following resources:

  • Read the latest documentation for Fluo and Fluo Recipes
  • Run the Fluo quickstart example application
  • View the phrase count example to see a more comprehensive Fluo application
  • See how you can get involved in the Fluo project
